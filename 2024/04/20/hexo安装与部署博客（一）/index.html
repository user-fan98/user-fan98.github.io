hexo安装与部署博客（一）

==博客的搭建完全按照大鱼的博客教程进行，仅有文末的404问题其教程中未提及，应该是新版本Github默认设置更改的原因。==

Hexo安装

  • 安装git客户端

    可直接从Git网页下载，windows安装完成后，git bash界面优于自带的cmd界面。通过==git –version==可以查看是否安装成功。

  • Hexo安装

    部署博客需要的Hexo基于Node.js(官网可直接下载安装)开发，是一个快速、高效、简洁的博客框架。其使用Markdown（一种轻量级标记语言）解析文章，并生成静态页面，非常便于静态部署和加载。

    安装Node后可通过==node -v== ==npm -v== 查看是否安装成功。

    Hexo开发社区非常活跃，有很多可选插件，能够实现包括自动部署、自动提交搜索引擎、自动提交图床等大量自动化功能。

    • 选择一个存储盘，建立一个博客专用的文件夹，在文件夹内打开git bash(cd /X/Blog)。输入==npm install hexo-cli -g==安装Hexo。同上==Hexo -v==可查看结果。
    • 在博客文件夹内打开命令终端，输入==hexo init== 初始化文件夹。初始化后的文件夹内容主要为配置文件。
    • 初始化完成后==hexo g== 生成静态文件，==hexo s== 打开本地服务器。本地网页预览如下。

Github部署

  • GitHub Pages是GitHub提供的一种免费的静态网页托管服务，首先需要注册github账号，且创建一个库（New repository），具体过程可参考廖雪峰的网站。

    需要==注意==的是，创建的repository名称要和用户名相同，例如Github账号用户名为qwert，则仓库名称应为qwert.github.io，这样博客部署到Github Page上时，才能够有效识别，即qwert.github.io。

  • 此后要将SSH添加到Github，链接Github与本地。输入命令：

    ==git config –global user.name “yourname”==（Github用户名）
    ==git config –global user.email “youremail”==（Github邮箱）

    输入完成后可以用：
    ==git config user.name==
    ==git config user.email==
    来检测是否输入正确。

  • 创建SSH，==ssh-keygen -t rsa -C “youremail”==

    SSH，可以理解为一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

    在用户主目录里找到.ssh目录，里面可以看到id_rsa和id_rsa.pub两个文件，前者是私钥，不能泄露。后者是公钥。

  • 打开Github，settings，此后选择SSH and GPG keys，新建一个SSH，名字随意，将id_rsa.pub复制进去，如下图。

    ![部署密钥过程](deploy keys.png)

此后在git bash输入ssh -T git@github.com，如果如下图所示，出现你的用户名，那就成功了。

  • 配置文件修改
    打开博客根目录下_config.yml文件，修改最后一行配置：

    –Deployment
    —-Docs: https://hexo.io/docs/one-command-deployment
    deploy:
    type: git
    repository: https://github.com/user-fan98/user-fan98.github.io.git
    branch: master

    ==注意== 此处“：”后需要空一行，最后branch修改为master，且deploy下的三行配置要保持缩进一致，否则后续部署博客会失败。

  • 博客部署

    首先需要安装deploy-git，即部署用的命令插件？
    npm install hexo-deploy-git –save

    hexo clean (清除之前写的内容，可不加)
    hexo g (生成静态文章)
    hexo d (部署文章)

==需要注意的是，在新版的Github中，需要更改settings中pages页面里的选项，否则博客网址会一直显示404==

Github现在版本默认为none，如我们使用的master不符，因此要特意更改一下。

  • 然后输入hexo new post “名称”，新建一篇文章。

    在你的博客根目录\source_posts的目录，可以发现下面多了一个文件夹和一个.md文件，一个用来存放你的图片等数据，另一个就是你的文章文件。==（这部分需另外参考一下插入图片设置，否则只会出现.md文件）==
    markdown文件前文有提及，本文使用的Typora。
    编写完markdown文件后，根目录下输入hexo g生成静态网页，然后输入hexo s可以本地预览效果，最后输入hexo d上传到github上。这时打开你的github.io主页就能看到发布的文章。

此文为基于Hexo+Github搭建博客的操作过程，按照网络教程操作即可，没有出现很Bug的情况，唯一的问题即最后提到的404问题，作为第一篇文字内容记录一下此过程。但是生成的文章格式还有很多问题，以及添加图片无法在网页显示的问题。
后续继续改进一下。
